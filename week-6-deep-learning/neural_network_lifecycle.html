Neural Network — Complete Lifecycle With Real Numbers

Every phase explained with data: Input → Forward Pass → Loss → Backpropagation → Weight Update → Convergence → Stop

OVERVIEW What Is Happening Here? (Read First)
We are training a small neural network to predict if a patient is diabetic (1) or not (0), given two inputs: glucose level and BMI.

The network starts with random weights and knows nothing. We show it one sample, it makes a wrong guess, we calculate how wrong it was, trace back to find which weights caused the error, nudge those weights slightly, and repeat thousands of times until predictions become accurate.

This entire process — one guess → measure error → fix weights → repeat — IS what "training a neural network" means.

Phase 1 — Forward Propagation  (Left → Right: The Network Makes Its Guess)

Input data flows layer by layer. Each neuron does two things: (1) weighted sum, (2) activation function. Result = prediction ŷ.

In forward propagation, we are NOT learning yet. We are just asking: "With the current weights, what does the network predict?" The network starts with random weights, so the first prediction will be wrong — that is expected and normal.
STEP 1 Hidden Neuron H1 — Receives From Both x1 and x2
STEP A — Weighted Sum (z_H1): ↳ Each input is multiplied by its OWN weight, then all are added together plus bias z_H1 = (w11 × x1) + (w12 × x2) + bias_H1 = (0.3 × 0.5) + (0.5 × 0.8) + 0.1 = 0.15 + 0.40 + 0.10 = 0.65 ↳ z_H1 = 0.65 means: "the raw signal strength arriving at H1 before activation" ↳ We SAVE z_H1 = 0.65 ← this value is needed again during backward pass STEP B — ReLU Activation: ↳ ReLU decides: should this neuron "fire" or stay silent? H1_output = max(0, z_H1) = max(0, 0.65) = 0.65 ✅ positive → neuron fires, passes through unchanged ↳ If z_H1 were -0.3, output = max(0, -0.3) = 0 → neuron stays silent (dead neuron)
STEP 2 Hidden Neuron H2 — Same Process, Different Weights
STEP A — Weighted Sum (z_H2): ↳ H2 has its OWN weights w21 and w22 — completely independent of H1's weights ↳ Both H1 and H2 receive the SAME inputs x1 and x2, but different weights make them learn different patterns z_H2 = (w21 × x1) + (w22 × x2) + bias_H2 = (0.2 × 0.5) + (0.4 × 0.8) + 0.2 = 0.10 + 0.32 + 0.20 = 0.62 ↳ We SAVE z_H2 = 0.62 ← needed during backward pass ReLU gate check STEP B — ReLU Activation: H2_output = max(0, z_H2) = max(0, 0.62) = 0.62 ✅ positive → passes through
Why do we "save" z values? During backward propagation, we need to check "was this neuron active during forward pass?" If z > 0 → it was active → gradient passes through. If z ≤ 0 → it was silent → gradient is blocked. The network stores all z values in memory during forward pass for use in backward pass.
STEP 3 Output Neuron O — Produces Final Prediction ŷ
STEP A — Weighted Sum (z_O): ↳ The output neuron receives H1_output and H2_output (not the original inputs x1, x2) ↳ It applies its own weights w31 and w32 to combine the hidden layer's outputs z_O = (w31 × H1_output) + (w32 × H2_output) + bias_O = (0.6 × 0.65 ) + (0.3 × 0.62 ) + 0.1 = 0.390 + 0.186 + 0.10 = 0.676 STEP B — Sigmoid Activation (used ONLY at output for binary classification): ↳ Sigmoid squashes ANY number into range (0, 1) — perfect for expressing probability ↳ Formula: 1 / (1 + e^(-z)) where e ≈ 2.718 (Euler's number) ŷ = 1 / (1 + e(-0.676)) = 1 / (1 + 0.509) = 1 / 1.509 = 0.663 ↳ ŷ = 0.663 means the model is 66.3% confident this person is diabetic ↳ Decision rule: if ŷ > 0.5 → predict "diabetic (1)", else → predict "healthy (0)" ↳ So model DOES predict diabetic — but with only 66.3% confidence, when real answer is 100% (y=1)

Phase 2 — Loss Function  (The Report Card: How Wrong Are We?)

Loss = a single number measuring the gap between prediction ŷ and real label y. The entire goal of training = make this number as small as possible.

Think of loss as a score on an exam. Loss = 0 means perfect. Loss = 2.0 means terrible. After each batch of records, we compute the average loss across the batch, then backpropagate to reduce it. The loss function is the "north star" — everything the network does is aimed at minimising it.
PHASE 2 Binary Cross-Entropy Loss Calculation
We use Binary Cross-Entropy for binary classification (0 or 1 output). It punishes confident wrong answers very heavily — much harder than MSE. If the model says 99% confident but is wrong → loss is huge (4.6). If model says 51% → loss is small (0.67).
Formula: L = -[ y × log(ŷ) + (1-y) × log(1-ŷ) ] ↳ Two terms: first handles when real label is 1, second handles when real label is 0 ↳ Since y = 1 here: (1-y) = 0, so the entire second term = 0 and disappears L = -[ 1 × log(0.663) + 0 × log(1-0.663) ] = -[ 1 × log(0.663) + 0 ] = -[ log(0.663) ] = -[ -0.411 ] = 0.411 ↳ log(0.663) is negative because log of any number less than 1 is negative ↳ The minus sign in front flips it to positive → loss is always ≥ 0
Interpreting loss values:    ŷ = 1.000 (perfect) → Loss = 0.000  ✅    ŷ = 0.900 (close)   → Loss = 0.105    ŷ = 0.663 (our case) → Loss = 0.411    ŷ = 0.100 (very wrong) → Loss = 2.303  ❌

Phase 3 — Backward Propagation  (Right → Left: Tracing the Blame)

We now ask: "Which weights caused this loss of 0.411, and by exactly how much?" Backprop answers this for every single weight using the Chain Rule.

The Chain Rule in plain English: Loss depends on the output. Output depends on hidden layer. Hidden layer depends on input weights. To find how much w11 caused the loss, we multiply the gradients at each layer together — like passing a blame signal backwards through a chain of people.
STEP 3A Output Layer — Blame Calculated First (Closest to Loss)
Output blame signal δ_O: ↳ Shortcut formula for Binary Cross-Entropy + Sigmoid (saves many calculation steps) ↳ This formula is derived by combining the chain rule for CE loss and sigmoid derivative δ_O = ŷ - y = 0.663 - 1.0 = -0.337 ↳ Negative value (-0.337) means: prediction was BELOW the target ↳ The network needs to push output HIGHER → so weights will INCREASE after update ↳ If δ_O were positive, weights would decrease. Sign of δ tells direction of correction. Now compute gradient for each output weight: ↳ Formula: gradient = blame_signal × (the input this weight was multiplying) ↳ Reasoning: a weight that multiplied a bigger value is MORE responsible for the output ∂L/∂w31 = δ_O × H1_output = -0.337 × 0.65 = -0.219 ∂L/∂w32 = δ_O × H2_output = -0.337 × 0.62 = -0.209 ∂L/∂b_O = δ_O × 1 = -0.337 × 1 = -0.337 ↳ bias gradient = δ_O × 1, because bias is always multiplied by 1 (it is a constant)
STEP 3B Hidden Layer — Blame Passes Backwards Through Chain Rule
Pass blame backwards to each hidden neuron: ↳ Each hidden neuron's blame = output blame × the weight that connected it to output ↳ H1 is connected to output via w31=0.6 (larger weight → more blame) ↳ H2 is connected to output via w32=0.3 (smaller weight → less blame) Blame at H1 = δ_O × w31 = -0.337 × 0.6 = -0.2022 Blame at H2 = δ_O × w32 = -0.337 × 0.3 = -0.1011 Apply ReLU Gate Check (the backward ReLU rule): ↳ ReLU was used in forward pass → must apply its derivative in backward pass ↳ ReLU derivative: 1 if z > 0 (gate was open) | 0 if z ≤ 0 (gate was closed = dead neuron) ↳ We check the z values we SAVED during forward pass H1: z_H1 = 0.65 > 0 → ReLU gradient = 1 → δ_H1 = -0.2022 × 1 = -0.2022 ✅ passes through H2: z_H2 = 0.62 > 0 → ReLU gradient = 1 → δ_H2 = -0.1011 × 1 = -0.1011 ✅ passes through ↳ If z were negative (e.g., z=-0.3): ReLU gradient = 0 → δ = 0 → w11,w12 never update = "dead neuron" ↳ This is why ReLU avoids vanishing gradient: gradient = 1, not a fraction like Sigmoid (max 0.25) Compute gradient for each hidden weight: ↳ Same formula: gradient = blame_signal × (input to that weight) ∂L/∂w11 = δ_H1 × x1 = -0.2022 × 0.5 = -0.1011 ∂L/∂w12 = δ_H1 × x2 = -0.2022 × 0.8 = -0.1618 ∂L/∂b_H1 = δ_H1 × 1 = -0.2022 × 1 = -0.2022 ∂L/∂w21 = δ_H2 × x1 = -0.1011 × 0.5 = -0.0506 ∂L/∂w22 = δ_H2 × x2 = -0.1011 × 0.8 = -0.0809 ∂L/∂b_H2 = δ_H2 × 1 = -0.1011 × 1 = -0.1011

🛑 When and Why Does Backpropagation Stop?

  • Backprop stops at the Input Layer — because x1 and x2 are your raw data (patient's glucose, BMI). You cannot change real-world data.
  • Only weights and biases inside the network are updated — these are the network's "knowledge".
  • We have now computed gradients for all 9 parameters (w11, w12, b_H1, w21, w22, b_H2, w31, w32, b_O).
  • Backprop is complete. Next step: use these gradients to update all 9 weights.

Phase 4 — Weight Update  (Gradient Descent: Apply the Corrections)

Now use the gradients computed in backprop to nudge every weight slightly in the direction that reduces loss.

Key insight about the sign: A negative gradient means increasing this weight would REDUCE the loss → so we increase it. A positive gradient means increasing this weight would INCREASE the loss → so we decrease it. The minus sign in the formula automatically handles both cases: W_new = W_old − (η × gradient)
PHASE 4 Weight Update Formula Applied to All 9 Parameters
Formula: W_new = W_old − (η × gradient) ↳ η = learning rate = 0.1 (your choice, set before training starts) ↳ Negative gradient → W_new > W_old (weight increases) — needed here because ŷ was too low ↳ Positive gradient → W_new < W_old (weight decreases) — used when prediction is too high ── OUTPUT LAYER WEIGHTS ── ↳ These sit between hidden layer and output — updated first because they're closest to the loss w31: 0.600 − (0.1 × -0.219) = 0.600 + 0.0219 = 0.6219 ↑ increased w32: 0.300 − (0.1 × -0.209) = 0.300 + 0.0209 = 0.3209 ↑ increased b_O: 0.100 − (0.1 × -0.337) = 0.100 + 0.0337 = 0.1337 ↑ increased ── HIDDEN LAYER WEIGHTS (H1) ── ↳ These sit between input layer and hidden layer H1 w11: 0.300 − (0.1 × -0.1011) = 0.300 + 0.0101 = 0.3101 ↑ w12: 0.500 − (0.1 × -0.1618) = 0.500 + 0.0162 = 0.5162 ↑ b_H1: 0.100 − (0.1 × -0.2022) = 0.100 + 0.0202 = 0.1202 ↑ ── HIDDEN LAYER WEIGHTS (H2) ── ↳ These sit between input layer and hidden layer H2 w21: 0.200 − (0.1 × -0.0506) = 0.200 + 0.0051 = 0.2051 ↑ w22: 0.400 − (0.1 × -0.0809) = 0.400 + 0.0081 = 0.4081 ↑ b_H2: 0.200 − (0.1 × -0.1011) = 0.200 + 0.0101 = 0.2101 ↑ ↳ All 9 parameters increased — because the model was predicting too low (0.663 < 1.0) ↳ The updates are tiny (in thousandths) — that is intentional. Gradual small steps = stable learning.
VERIFY Run Forward Pass Again With New Weights — Did Loss Improve?
After every weight update, we can verify that the loss actually decreased by running one more forward pass with the new weights. This is not done in practice after every sample (too slow), but it proves the math works.
Forward pass with NEW weights: z_H1 = (0.3101×0.5) + (0.5162×0.8) + 0.1202 = 0.1551 + 0.4130 + 0.1202 = 0.6883 H1 = ReLU(0.6883) = 0.6883 z_H2 = (0.2051×0.5) + (0.4081×0.8) + 0.2101 = 0.1026 + 0.3265 + 0.2101 = 0.6392 H2 = ReLU(0.6392) = 0.6392 z_O = (0.6219×0.6883) + (0.3209×0.6392) + 0.1337 = 0.4282 + 0.2051 + 0.1337 = 0.7670 ŷ = Sigmoid(0.7670) = 0.683 ↳ New loss = -log(0.683) = 0.381 ┌─────────────────────────────────────────────────────────┐ │ Before update: ŷ = 0.663 → Loss = 0.411 │ │ After update: ŷ = 0.683 → Loss = 0.381 ↓ improved ✅ │ └─────────────────────────────────────────────────────────┘ ↳ Loss dropped by 0.030 after just ONE update on ONE sample ↳ Repeat this across 700 records × 300 epochs = hundreds of thousands of updates ↳ Each update is tiny, but compounded over thousands of iterations → loss converges near 0
How close to perfect after many epochs?
After ~300 epochs on 700 records:   ŷ ≈ 0.92 for diabetic patients  |  ŷ ≈ 0.08 for healthy patients  |  Accuracy ≈ 85–92% (typical for this type of problem)

Phase 5 — Training Loop & Stopping Conditions

How 700 records are processed in batches across many epochs, and exactly when and why training stops.

YOUR DATA How Your 700 Records Are Processed
Common confusion: Is loss calculated for each of the 700 records separately? Answer: No — for mini-batch gradient descent (the standard method), records are grouped into batches of 32. Loss is the AVERAGE across all 32 records in the batch. One backprop runs per batch, not per record.
Mini-Batch processing for 700 records, batch size = 32: ↳ 700 ÷ 32 = 21.875 → rounded to 22 batches per epoch (last batch may have fewer records) Epoch 1: Shuffle 700 records randomly ↳ Shuffling prevents the network from memorising the order of records Batch 1 (records 1–32): 32 forward passes → average loss → 1 backprop → update all 9 weights Batch 2 (records 33–64): 32 forward passes → average loss → 1 backprop → update all 9 weights Batch 3 (records 65–96): ... ... Batch 22 (records 673–700): remaining records → average loss → 1 backprop → update all 9 weights ↳ End of Epoch 1: weights updated 22 times. Model has seen all 700 records once. Epoch 2: Shuffle again → repeat same 22 batches with slightly better weights ↳ Same records, different order. The model sees the same data many times but from different angles. Epoch 300: Weights have been updated: 22 batches × 300 epochs = 6,600 total weight updates Loss has converged from ~0.69 (random) down to ~0.08 (trained) ✅
STOPPING 3 Conditions That Stop Training
① Epoch Limit (epoch=300)
Simplest stop. After 300 epochs, training ends.
Risk: may stop too early (underfitting) or too late (overfitting). Use with early stopping for safety.
② Early Stopping ← Best Practice
Monitor validation loss after each epoch.
If val_loss doesn't improve for patience=20 consecutive epochs → stop.
Automatically restore best weights from before the drift.
Best for small datasets like 700 records.
③ Loss Threshold
Stop when loss drops below a defined threshold (e.g., loss < 0.05).
Rarely used alone — early stopping is more robust.

What is validation loss? Before training, split data into training (80%) and validation (20%). For 700 records: 560 used for training, 140 held out for validation. After each epoch, run forward pass on the 140 validation records (no weight updates), compute their loss. If that loss rises while training loss falls → model is memorising, not learning → time to stop.
OVERFITTING Training Loss vs Validation Loss — Recognising When to Stop
Overfitting (too many epochs):
Train loss: keeps falling →
Val loss: starts rising →
GAP between them grows →
Model memorised training data, fails on new data.
Stop here. Use early stopping.
Good fit (correct epochs):
Both train loss and val loss falling together →
Gap stays small →
Model has genuinely learned the pattern →
Predicts well on new, unseen data too.
This is the goal.
Recommended settings for 700 records:   epochs=300, batch_size=32, validation_split=0.2, patience=20, restore_best_weights=True
Expected actual stopping: around epoch 150–200 (early stopping fires before hitting epoch limit).

FORWARD Forward Pass Formulas
z = Σ(weight × input) + bias ↳ sum of all weighted inputs + bias ReLU: max(0, z) ↳ hidden layers — gradient = 1 if z>0 Sigmoid: 1 / (1 + e⁻ᶻ) ↳ binary output only — range (0,1) Softmax: eᶻⁱ / Σeᶻʲ ↳ multi-class output — sums to 1.0
LOSS Loss Functions
Binary CE: -[y·log(ŷ)+(1-y)·log(1-ŷ)] ↳ binary classification (0 or 1) MSE: (1/n) × Σ(y - ŷ)² ↳ regression (predicting numbers) MAE: (1/n) × Σ|y - ŷ| ↳ regression, robust to outliers Categorical CE: -Σ yᵢ × log(ŷᵢ) ↳ multi-class classification
UPDATE Weight Update Rule
W_new = W_old − η × gradient ↳ η = learning rate (your choice) ↳ negative gradient → weight increases ↳ positive gradient → weight decreases ↳ Adam default η = 0.001
BACKPROP Backward Pass Formulas
Output blame (CE + Sigmoid): δ_O = ŷ - y ↳ negative = prediction too low Hidden blame (chain rule): blame_H = δ_O × connecting_weight ↳ bigger weight = more blame Through ReLU gate: δ_H = blame_H × (1 if z>0 else 0) ↳ checks saved z from forward pass Weight gradient: ∂L/∂w = δ × input_to_that_weight ↳ bigger input = bigger gradient Bias gradient: ∂L/∂b = δ × 1 = δ ↳ bias always multiplied by 1
WHY RELU ReLU vs Sigmoid Gradient
Sigmoid gradient: max = 0.25 4 layers: 1 × 0.25⁴ = 0.004 → vanishes! ↳ early layers get almost zero gradient ReLU gradient: = 1 (when z > 0) 4 layers: 1 × 1⁴ = 1.0 → full signal ↳ all layers get proper gradient
PARAMS Counting Parameters
Weights = left neurons × right neurons ↳ every connection has 1 weight Input(2)→Hidden(2): 2×2 = 4 weights Hidden(2)→Output(1): 2×1 = 2 weights Biases: 1 per neuron = 3 biases Total = 9 parameters
GLOSSARY Every Term — Plain English, One Line Each
TermPlain English Meaning
NeuronOne computing unit inside the network. Does: weighted sum → activation function → output.
WeightOne number per connection (arrow). Controls how strongly one neuron influences the next. Learned by backprop.
BiasOne number per neuron. Allows neuron to fire even when all inputs are zero. Like the y-intercept in y = mx + c.
ReLUActivation: max(0, z). Passes positive values unchanged. Blocks negatives. Gradient = 1 → no vanishing gradient.
SigmoidActivation: squashes any number to (0,1). Used at output for binary classification. Gradient max 0.25 → vanishes in deep nets.
Forward PassData flows left→right through all layers. Each layer: z = Σ(wx)+b, then activation. Produces prediction ŷ.
LossSingle number: how wrong the prediction is. Loss = 0 → perfect. Loss = 2.3 → terrible. Goal: minimise this.
Gradient"If I increase this weight by a tiny amount, does loss go up or down, and by how much?" — the answer is the gradient.
BackpropagationTraces loss backwards right→left. Uses chain rule to assign a gradient (blame) to every weight and bias.
Chain RuleEach layer multiplies the incoming blame by its own local gradient and passes it further left.
Dead Neuronz ≤ 0 during forward pass → ReLU output = 0 → backprop gradient = 0 → weights never update → neuron stuck forever.
Vanishing GradientSigmoid multiplies gradient by ≤0.25 per layer. Deep networks multiply many times → gradient ≈ 0 → early layers don't learn.
Weight UpdateW_new = W_old − η × gradient. Nudges every weight in the direction that reduces loss. Runs after every batch.
Learning Rate ηStep size for weight update. Set by YOU before training. Not learned. Too large = overshoot. Too small = too slow.
EpochOne full pass through all training records. 700 records ÷ batch 32 = 22 weight updates per epoch.
BatchGroup of records processed together. Loss = average across batch. One backprop per batch.
Early StoppingWatches validation loss. If it doesn't improve for patience=20 epochs → stop and restore best weights.
OverfittingTrain loss falls but val loss rises. Model memorised training data. Fix: early stopping, dropout, more data.
